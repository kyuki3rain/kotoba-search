# AGENTS guide for kotoba-search

- **回答は日本語**: 内部の思考やメモは英語でも構わないが、ユーザーに返すメッセージは常に日本語でまとめる。丁寧かつ簡潔に。必要があれば絶対日時も明示する。
- **YAGNI を徹底**: 追加実装や依存は「必要になってから」。要求に含まれない装飾や機能は避け、最小構成を保つ。
- **迷ったら先に相談**: 仕様・依存・データなど不明点があれば、実装に入る前に調査結果を共有し、即座にユーザーへ確認を取る。勝手な推測で進めない。
- **ビルド／サーブ**: 語彙生成は `mise run build_words`（Python スクリプト）、ローカル確認は `mise run serve`。他のツールは不要。
- **Chrome DevTools MCP**: GUI 検証が必要なときは `chrome_devtools` サーバーを利用してブラウザを制御できる。既存プロセスが残っていないか確認する。
- **ライセンス配布物**: `public/COPYING-ipadic.txt` と `public/NOTICE.txt` は `build_words` が更新する。手動編集は避ける。
- **語彙ポリシー**: mecab-ipadic の読みをひらがな化したリストを配布。UI の説明やプレースホルダもひらがなモード前提で維持する。
- **バージョン管理の慎重さ**: ユーザー環境は Git 未初期化の場合がある。破壊的コマンド（reset 等）は厳禁。既存ファイルの非 ASCII には注意。
